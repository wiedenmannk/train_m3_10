HTTP-Statuscodes

Wenn ein Browser eine Anfrage stellt, antwortet der Server mit einem Statuscode.

Browser fragt → Server antwortet

Was sagt ein Statuscode?

Ein Statuscode beschreibt, ob eine Anfrage erfolgreich war oder ob ein Problem aufgetreten ist.

Der Statuscode ist die kurze Antwort des Servers.

Die wichtigsten Codes

200

OK

Die Seite wurde erfolgreich geladen.

400

Bad Request

Die Anfrage war fehlerhaft oder unverständlich.

403

Forbidden

Der Zugriff ist verboten.

404

Not Found

Die angeforderte Seite wurde nicht gefunden.

500

Server Error

Auf dem Server ist ein Fehler passiert.

400 Bad Request

Der Server versteht die Anfrage nicht richtig.

Beispiel: Ein Formular sendet ungültige oder fehlende Daten.

Merksatz: Die Anfrage ist kaputt, nicht unbedingt der Server.

Mini-Beispiele

  • 200: Startseite lädt normal.
  • 400: Formulardaten sind unvollständig oder fehlerhaft.
  • 403: Zugriff auf Adminbereich ohne Berechtigung.
  • 404: Falsche URL eingegeben.
  • 500: Fehler im Servercode.

Wie sieht das im Log aus?

Ein Server-Log kann stark vereinfacht so aussehen:

GET /index.php 200

POST /save.php 400

GET /admin.php 403

GET /seite-gibt-es-nicht.php 404

POST /login.php 500

Muster: Methode → angefragte Seite → Statuscode

Pizza-Merksatz

  • 200: Pizza kommt.
  • 400: Bestellung ist unverständlich.
  • 403: Du darfst nicht bestellen.
  • 404: Restaurant gibt es nicht.
  • 500: Küche brennt.
