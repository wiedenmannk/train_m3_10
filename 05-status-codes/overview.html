🌐 HTTP Statuscodes – verstehen statt merken

2xx – Erfolg

200 OK → Seite wurde erfolgreich geladen
201 Created → etwas wurde neu erstellt (z. B. Datensatz)
204 No Content → erfolgreich, aber keine Antwort nötig
GET /index.html → 200

4xx – Client Fehler

400 Bad Request → Anfrage falsch (z. B. falsche Daten)
401 Unauthorized → nicht eingeloggt
403 Forbidden → keine Berechtigung
404 Not Found → Seite existiert nicht
GET /admin → 401

5xx – Server Fehler

500 Internal Server Error → Fehler im eigenen Server (z. B. PHP crash)
502 Bad Gateway → Server bekommt falsche Antwort von anderem Server
503 Service Unavailable → Server ist überlastet oder down
POST /save → 500

🔥 Wichtige Unterschiede (Prüfungsfalle!)

500 vs 502 vs 503:

500 → Fehler im eigenen Code
👉 Beispiel: PHP Script kaputt

502 → Fehler bei Kommunikation mit anderem Server
👉 Beispiel: Backend antwortet falsch

503 → Server funktioniert, aber ist überlastet / nicht verfügbar
👉 Beispiel: Wartung oder zu viele Nutzer
401 vs 403:

401 → „Wer bist du?“ (Login fehlt)
403 → „Ich kenne dich, aber du darfst nicht rein“
