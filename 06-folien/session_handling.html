Session & Header – Wie erkennt der Server dich?

1. Login
Server erstellt eine Session
Session-ID: abc123
2. Server antwortet (Response-Header)
Set-Cookie: PHPSESSID=abc123
3. Browser speichert Cookie
PHPSESSID = abc123
4. Nächster Request (Request-Header)
GET /dashboard.php HTTP/1.1 Cookie: PHPSESSID=abc123
5. Server erkennt dich wieder
abc123 → User Klaus
💡 Merksatz:
Die Session liegt auf dem Server.
Der Browser bringt nur den Schlüssel (Session-ID) mit.
✔ Beispiel Log (normal):
POST /login.php 200 GET /dashboard.php 200
⚠ Beispiel Problem:
Kein Cookie → neue Session
GET /dashboard.php 200
